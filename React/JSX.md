# JSX 란 무엇인가? !!!
**JSX(JavaScript XML)** 은 자바스크립트 코드안에서 HTML과 유사한 마크업을 직접 작성할수 있게 해주는 문법 확장 입니다!
브라우저는 JSX를 직접 읽을 수 없지만, 빌드도구(Vite,Babel 등)가 이를 자바스크립트 코드로 변환 해줍니다.

코드비교 : JSX vs 일반 js
JSX를 쓰면 코드가 얼마나 직관적으로 변환지 봐보죠!

* **JSX 사용 시 (우리가 작성하는 코드)** <br>
```const element = <h1 className="title">안녕하세요, 남규님!</h1>;```

* **일반 JS 변환시 (브라우저가 실행하는 코드)** <br>
``` const element = React.createElement('h1', {className: 'title'}, '안녕하세요, 남규님!'); ```

<br>

# JSX 사용해야하는 이유

* **가독성과 생산성**: HTML과 구조가 비슷해 UI의 형태를 한눈에 파악할 수 있고, 별도의 HTML 파일을 오갈 필요가 업습니다.
* **강력한 기능**: 자바스크립트의 모든 문법(변수,함수,루프 등)을 UI 작성 과정에서 그대로 사용할수 있습니다.
* **보안**: 리액트는 렌더링 전 JSX에 삽입된 모든 값을 문자열로 변환하여 **XSS(교차 사이트 스크립팅)** 공격을 방지합니다.

<br>

# JSX 작성 시 꼭 지켜야 할 핵심 규칙

실무에서 에러를 방지하기 위해 반드시 기억해야 할 3가지 규칙입니다.

① 반드시 하나의 부모 요소로 감싸야 함
여러 요소가 나열될 때는 반드시 하나의 태그(예: <div> 또는 빈 태그 <> </>)로 묶어야 합니다.

```
// 잘못된 예
return (
  <h1>제목</h1>
  <p>내용</p>
)

// 올바른 예 (Fragment 사용)
return (
  <>
    <h1>제목</h1>
    <p>내용</p>
  </>
)
```

<br>

② CamelCase 스타일의 속성명 사용
JSX는 자바스크립트에 가깝기 때문에 HTML 속성명이 자바스크립트 예약어와 겹치지 않도록 CamelCase를 사용합니다.

* class → className

* onclick → onClick

* tabindex → tabIndex

③ 중괄호 { }를 사용한 자바스크립트 표현식 삽입
변수나 함수 결과를 화면에 보여주고 싶을 때는 중괄호 안에 작성합니다.

```
const name = "남규";
return <h1>{name}님, 힘내세요!</h1>;

```
